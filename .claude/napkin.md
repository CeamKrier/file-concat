## Patterns That Work

- For SSR-safe browser-only state, guard `useState(initialiser)` and any helper called from a state initialiser with `if (typeof window === "undefined") return <fallback>`. `useEffect` callbacks already only run on the client, so localStorage reads inside an effect are fine without the guard.
- For TanStack Start in apps/web (with `@cloudflare/vite-plugin` v1.22+), the SSR build outputs to `apps/web/dist/server/index.js` and static assets to `apps/web/dist/client/`. Run the worker with `pnpm start` (= `node dist/server/index.js`). The `.output/` convention is from older TanStack Start versions and no longer applies ŌĆö keep both `dist/` and `.output/` in `.gitignore`.
- **Live-reactive filter via validations cache + override layer + derived effect.** For an interactive surface where ingestion-time validation is expensive but pattern matching is cheap: build `validations: Record<path, ValidationRecord>` once at ingest (binary/size check), keep `userToggled: Record<path, "include"|"exclude">` as a sticky override map, then a single `useEffect` deriving `fileStatuses.included` from `(validations, patterns, userToggled)` whenever any dependency changes. Manual toggles write to the override map (not directly to status), so pattern edits never wipe a user's explicit decision. Lives in `apps/web/src/app.tsx`.
- **Catalog over raw API for canonicalized lists.** `models.dev` exposes `api.json` (per-provider raw, 4757 entries with massive duplication), `models.json` (canonical metadata only, no cost), and `catalog.json` (combined: 203 canonical models + provider offerings with cost). We use `catalog.json` and `buildTextModelsFromCatalog` in `@fileconcat/core`. Per-canonical-model we pick the cheapest priced provider; the `FilteredModel.uid` is the catalog id (`lab/model-id`) so it stays stable when the cheapest provider shifts across deploys.
- **`Omit<Type, "field">` to express server-vs-client payload divergence.** `ModelsRegistry.providers` is optional and dropped from both the bundled `models.json` and the `/api/models` response because the runtime never reads it. Avoids 4MB+ of unread JSON in every page load.

## Measured Findings

- **`removeEmptyLines` was a token-cost INCREASE, not decrease.** Measured 2026-06-07 with tiktoken cl100k_base across 8 representative repo files: total +5.93% (+1047 tokens). Zero files showed savings. CRLF endings + the `^\s*[\r\n]/gm` regex appear to interact badly with the tokenizer. Feature removed end-to-end in this commit. Do not reintroduce a "remove blank lines" toggle without re-measuring against CRLF input.
- **`addLineNumbers` costs ~+44.83% tokens.** Expected, this is a navigation aid not a saving. UI now labels it accordingly in `filter-rail.tsx`.

## Patterns That Don't Work

- `minimatch(fullPath, "node_modules", { dot: true })` does NOT match `"node_modules/react/index.js"`. minimatch matches the entire path against the pattern; bare directory names match only the exact string. To honour bare-name ignore patterns, also test each path segment individually (`path.split("/").some(seg => minimatch(seg, pattern))`).
- Vite's `import.meta.glob("~/some/path/*.mdx")` does NOT resolve the `~` (or any) path alias in the glob pattern ŌĆö it returns an empty record and every keyed lookup fails. Use a relative path (`"../../content/foo/*.mdx"`) or an absolute path (`"/src/content/foo/*.mdx"`). The keys returned are in the same form you wrote the glob in, so the lookup string must match.
- Calling browser-only APIs (`localStorage`, `window`, `document`, `matchMedia`) at module top level OR inside a `useState(initialiser)` will crash SSR with `ReferenceError: X is not defined`. Always guard.
- Vitest's `environment: "jsdom"` does NOT give you a fully working `localStorage` under Node 22+ ŌĆö Node's experimental `--localstorage-file` builtin shadows it as a stub that lacks `.clear()`. Fix in test setup: `Object.defineProperty(window, "localStorage", { configurable: true, value: new MemoryStorageImpl() })` (and mirror onto `globalThis`). Same setup file must polyfill `window.matchMedia` unconditionally ŌĆö jsdom does not provide one, and any `useEffect` that calls it on mount will throw in tests.
- `minimatch(path, "src/**/*", { matchBase: true })` does NOT match `"myrepo/src/app.tsx"` ŌĆö `matchBase` is ignored when the pattern contains `/`, and a bare directory name does not match the same name sitting deeper in the path. **Fixed 2026-06-07** via `packages/core/src/path-utils/glob-match.ts`: `pathMatches(filePath, pattern)` handles "match anywhere" semantics ŌĆö bare patterns match any segment (plus `matchBase` for basename globs); slashed patterns try a full match plus every suffix obtained by stripping leading dir components. `matchesAnyPattern(filePath, list)` consumes the comma-separated form persisted in `UserConfig.{includePatterns,ignorePatterns}`. Both `isIgnoredDirectory` and `isExcludedPath` in `apps/web/src/app.tsx` now route through these helpers ŌĆö do not reintroduce inline `minimatch(...)` for user-supplied patterns.

## Domain Notes

- ESLint config (`eslint.config.js`) is minimal ŌĆö no `argsIgnorePattern` for `_`-prefixed names. So `const _foo = bar()` STILL trips `@typescript-eslint/no-unused-vars`. Either delete the assignment or actually consume it; the underscore prefix is not a ts-eslint escape hatch in this repo.
- Catch-clauses without a binding (`catch { ... }`) are the right way to silence unused-`error` lint, since the rule applies to caught errors here.
- `addLineNumbers` in `packages/core/src/file-processing/transform.ts` must drop a trailing empty element after `split("\n")` so a final newline doesn't produce a phantom numbered line. Tests in `tests/file-processing.test.ts` rely on this.
- **`/app` workbench architecture (post 2026-06-07 redesign):** three-region grid ŌĆö top `SourceBar` with project + counts + foreground "Start over"; left `FilterRail` (lg+) or Radix Dialog drawer (<lg) holding preset chips + include/ignore textareas + counter + Advanced disclosure; main column with `FileTree`, optional `TokenSection`; bottom-sticky `ActionBar` carrying token summary + Options popover (single/multi + xml/markdown + chunk size) + Copy + Download. State ownership stays in `app.tsx`. `WorkbenchHeader` + `ConfigPanel` + `PreviewModal` (was dead) are gone.
- **Drop-time vs filter-time pattern split:** at directory walk, only the hardcoded `HARDCODED_PRUNE_DIRS = {".git", "node_modules"}` set is excluded (browser-crash safety). Every other ignore / include pattern is purely post-ingestion via the live `useEffect`. Preset chip clicks therefore re-shape the already-dropped set instantly. Cost: a drop containing `dist/`, `build/`, `vendor/`, lock files etc. pulls them into memory before patterns hide them. Acceptable for typical projects; audit if drops feel slow.

The bundled `models.json` lives at `apps/web/src/data/models.json`.
