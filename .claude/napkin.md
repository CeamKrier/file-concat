## Patterns That Work

- For SSR-safe browser-only state, guard `useState(initialiser)` and any helper called from a state initialiser with `if (typeof window === "undefined") return <fallback>`. `useEffect` callbacks already only run on the client, so localStorage reads inside an effect are fine without the guard.
- For TanStack Start in apps/web (with `@cloudflare/vite-plugin` v1.22+), the SSR build outputs to `apps/web/dist/server/index.js` and static assets to `apps/web/dist/client/`. The SSR bundle is a **Cloudflare Worker module**, NOT a Node-runnable ESM entry â€” running it with `node dist/server/index.js` throws `Cannot read properties of undefined (reading 'compatibilityFlags')` because `globalThis.Cloudflare` only exists inside workerd. To run the built artifact locally, use `pnpm start` (= `wrangler dev --config dist/server/wrangler.json`), which spins up workerd locally with the generated config and the prebuilt assets. The `.output/` convention is from older TanStack Start versions and no longer applies â€” keep both `dist/` and `.output/` in `.gitignore`.
- **Live-reactive filter via validations cache + override layer + derived effect.** For an interactive surface where ingestion-time validation is expensive but pattern matching is cheap: build `validations: Record<path, ValidationRecord>` once at ingest (binary/size check), keep `userToggled: Record<path, "include"|"exclude">` as a sticky override map, then a single `useEffect` deriving `fileStatuses.included` from `(validations, patterns, userToggled)` whenever any dependency changes. Manual toggles write to the override map (not directly to status), so pattern edits never wipe a user's explicit decision. Lives in `apps/web/src/app.tsx`.
- **Catalog over raw API for canonicalized lists.** `models.dev` exposes `api.json` (per-provider raw, 4757 entries with massive duplication), `models.json` (canonical metadata only, no cost), and `catalog.json` (combined: 203 canonical models + provider offerings with cost). We use `catalog.json` and `buildTextModelsFromCatalog` in `@fileconcat/core`. Per-canonical-model we pick the cheapest priced provider; the `FilteredModel.uid` is the catalog id (`lab/model-id`) so it stays stable when the cheapest provider shifts across deploys.
- **`Omit<Type, "field">` to express server-vs-client payload divergence.** `ModelsRegistry.providers` is optional and dropped from both the bundled `models.json` and the `/api/models` response because the runtime never reads it. Avoids 4MB+ of unread JSON in every page load.
- **`import.meta.env.SSR` is the canonical Vite DCE guard for keeping a dependency out of the SSR worker.** Pattern: `if (import.meta.env.SSR) { return Promise.resolve(); } const m = await import("./client-only-thing");`. Vite replaces `import.meta.env.SSR` with a literal `true`/`false` at build time, then rollup DCE drops the unreachable branch and the dynamic import never enters the SSR bundle. `typeof window === "undefined"` is a runtime check and does NOT trigger DCE. Lives in `apps/web/src/lib/tokens.ts` as a facade in front of `tokens-client.ts`.

## Patterns That Don't Work

- `minimatch(fullPath, "node_modules", { dot: true })` does NOT match `"node_modules/react/index.js"`. minimatch matches the entire path against the pattern; bare directory names match only the exact string. To honour bare-name ignore patterns, also test each path segment individually (`path.split("/").some(seg => minimatch(seg, pattern))`).
- Vite's `import.meta.glob("~/some/path/*.mdx")` does NOT resolve the `~` (or any) path alias in the glob pattern â€” it returns an empty record and every keyed lookup fails. Use a relative path (`"../../content/foo/*.mdx"`) or an absolute path (`"/src/content/foo/*.mdx"`). The keys returned are in the same form you wrote the glob in, so the lookup string must match.
- Calling browser-only APIs (`localStorage`, `window`, `document`, `matchMedia`) at module top level OR inside a `useState(initialiser)` will crash SSR with `ReferenceError: X is not defined`. Always guard.
- Vitest's `environment: "jsdom"` does NOT give you a fully working `localStorage` under Node 22+ â€” Node's experimental `--localstorage-file` builtin shadows it as a stub that lacks `.clear()`. Fix in test setup: `Object.defineProperty(window, "localStorage", { configurable: true, value: new MemoryStorageImpl() })` (and mirror onto `globalThis`). Same setup file must polyfill `window.matchMedia` unconditionally â€” jsdom does not provide one, and any `useEffect` that calls it on mount will throw in tests.
- `minimatch(path, "src/**/*", { matchBase: true })` does NOT match `"myrepo/src/app.tsx"` â€” `matchBase` is ignored when the pattern contains `/`, and a bare directory name does not match the same name sitting deeper in the path. **Fixed 2026-06-07** via `packages/core/src/path-utils/glob-match.ts`: `pathMatches(filePath, pattern)` handles "match anywhere" semantics â€” bare patterns match any segment (plus `matchBase` for basename globs); slashed patterns try a full match plus every suffix obtained by stripping leading dir components. `matchesAnyPattern(filePath, list)` consumes the comma-separated form persisted in `UserConfig.{includePatterns,ignorePatterns}`. Both `isIgnoredDirectory` and `isExcludedPath` in `apps/web/src/app.tsx` now route through these helpers â€” do not reintroduce inline `minimatch(...)` for user-supplied patterns.
- **Radix `ScrollArea` defeats `truncate` on its children.** The viewport wraps content in an internal `<div style="display:table; min-width:100%">`. `display:table` shrink-wraps to `max-content`, so a row wider than the popover makes that wrapper grow past the popover width; the viewport's `overflow-x:hidden` then *hard-clips* it (no ellipsis, no scrollbar) and any `truncate` child renders at full natural width â€” pushing trailing content (e.g. the ctx/price in the model row) off-screen. Confirmed 2026-06-22 via Playwright: a 75-char name left `nameSpan.clientWidth === scrollWidth === 560px`, `ellipsisActive=false`, row 693px inside a 378px popover. Fix: for a list that needs per-row `truncate`, do NOT wrap it in `ScrollArea`; use a native `<div className="h-[...] overflow-y-auto overflow-x-hidden">`. The native block constrains children to the popover width so flex `min-w-0` + `truncate` engage (`ellipsisActive=true`, row holds at popover width). Also aligns with the product-register guidance against reinvented scrollbars. Lives in `apps/web/src/components/model-selector.tsx`.
- **`manualChunks: { foo: ["wasm-using-package"] }` combined with `vite-plugin-wasm` leaves an orphan stub in the SSR bundle even after DCE removes all real uses of the package.** The stub is whittled down to a single `import "./X_bg.wasm"` side-effect, which is enough for wrangler to attach the wasm as `compiled-wasm` and count it against the worker size limit. Even if `import.meta.env.SSR` DCE removes every JS reference to the package, the forced `manualChunks` entry recreates the side-effect-only chunk. Fix: drop the package from `manualChunks` entirely and let vite split it naturally on the client; if no client-reachable code imports it the chunk simply isn't emitted. Seen 2026-06-13 with `@dqbd/tiktoken` blocking the 3 MiB Workers free-tier limit.

- **Theme flash root cause (fixed 2026-06-13 in `routes/__root.tsx`):** the SSR HTML ships `<html>` with no theme class. BUT `ThemeProvider` (`theme-provider.tsx`) runs a `useEffect` that does `root.classList.remove("light","dark"); root.classList.add(theme)` on hydration. When theme resolves to `"light"`, that *adds* the `.light` class to `<html>` after the dropzone has painted. The dropzone uses `transition-[background-color,border-color,box-shadow,transform] duration-300`, so the CSS-variable jump triggered by that class addition animates as a 300ms flash (looks like whiteâ†’black to dark-mode users when the symptom is the dropzone re-settling). Fix: have the inline head script append a `<style data-theme-init>` containing `*,*::before,*::after { transition-duration:0s !important; transition-delay:0s !important; animation-duration:0s !important; animation-delay:0s !important; }` BEFORE adding the theme class, then remove the style after 2 `requestAnimationFrame`s post-`load`. Suppression covers the entire init window including the post-hydration `useEffect` flip. Do not collapse the double-rAF to a single one â€” the first frame is the layout settle, the second is the actual paint.
