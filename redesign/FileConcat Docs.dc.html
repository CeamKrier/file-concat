FileConcat docs

Everything FileConcat does, and how to bend it when you need to. The browser tool needs no setup; the CLI is a separate package, called out clearly where it appears.

Quick start

Drop a folder or files onto the page. The text-based files get read and combined, the noise gets skipped, and one document comes out — ready to copy or download. Nothing is uploaded.

What gets combined

Anything text-based is fair game. Binaries and build noise are filtered out automatically, so a whole folder can go in untouched.

Combined
Code · .js .ts .jsx .tsx .py .go .rs .java .rb .php .c .cpp
Docs · .md .txt .rst .adoc .org
Config · .json .yaml .toml .ini .env .xml
Data · .csv .tsv .sql
Skipped for you
Images, fonts, audio & video
Compiled binaries, archives
node_modules, vendor
Lockfiles & dist / build
i Images can't be combined — they aren't text. To have a model read pictures, use a vision-capable model, which is a different kind of tool.

Output formats

Three shapes for the same bundle. XML is the default — tags keep file boundaries unambiguous across long context. Switch any time; the token count updates live.

XML · default
<codebase project="app">
  <file path="src/index.ts" lang="ts">
    export const ok = true;
  </file>
</codebase>
Markdown
# app

## src/index.ts
```ts
export const ok = true;
```
Plain
===== src/index.ts =====
export const ok = true;

Large projects can be emitted as a single file or split into numbered parts — splitting just pastes more reliably into a chat, it changes nothing about the content.

Filtering & presets

Smart defaults handle the common case. When you want control, open Adjust what's included on the result screen.

PRESETSOne tap for React / Next, Vue, Python, Go, Rust — plus Source-only and Docs-only.
IGNOREComma-separated globs to skip. node_modules, *.lock, dist are already on by default.
INCLUDELeave empty for everything readable, or list globs to keep only those.
TREEUncheck any file or folder in the live tree to leave it out of the bundle.

Import by URL

No local files needed. Paste a link and FileConcat fetches it, then combines it the same way a drop would.

GitHub repos GitLab Bitbucket Gists Any web page 
https://github.com/owner/repo
https://gist.github.com/owner/id
https://example.com/docs/page

Zips unpack automatically. Web pages come in as their readable text, with nav and scripts stripped. If a link points at a non-text file — an image, a binary — it's flagged rather than combined.

Tokens & cost

Token counts are exact, computed locally with tiktoken — the same encoding the major models use. A rough cost estimate sits beside it so big bundles don't surprise you.

COUNTUpdates live as you include or exclude files.
COSTPick a model to see an estimate; an output-ratio control factors in the reply.
SPLITOver a model's context window, a multi-part split is suggested — as info, never a blocker.

Privacy

Files are read and combined entirely in your browser. There is no upload step, no account, and no server in the loop. After the page loads once, it works with the network off.

Separate tool · command line

CLI

A separate package from the browser app. Install it once and pipe directories straight into scripts — same structured output, printed to stdout. You don't need this for the web tool.

$ npm install -g @fileconcat/cli

$ file-concat ./src -o context.xml
$ file-concat ./docs --parse pdf,docx --json
-o <file>Write to a file instead of stdout.
--parseExtract text from PDF, DOCX, XLSX, PPTX and the ODF family.
--jsonPrint a one-line summary — counts, bytes, elapsed — for scripts.
-i / -eInclude / exclude globs, same idea as the browser tool.

FAQ

Can it combine images?

No — FileConcat works with text. Images aren't text, so there's nothing to concatenate. Reading pictures needs a vision model, a different kind of tool.

Is my code uploaded anywhere?

No. Everything happens in your browser — no upload, no server, no account.

What about a huge repo?

It still works. If it's over a model's context window, a multi-part split is offered to make pasting easier.

Can it read PDF or DOCX?

In the browser app these are treated as binaries and skipped. The CLI can extract their text with --parse.

Does it work offline?

Yes — once the page has loaded, you can turn the network off and keep going.
