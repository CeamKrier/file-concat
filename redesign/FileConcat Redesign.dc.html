What you get back

One file, structured so models read it cleanly.

Every file arrives wrapped and labeled — the project tree first, then each file in a tagged block with its path and language. The model walks the bundle the way you'd walk the folder.

TREEAn ASCII map at the top, so the model knows the shape before any file body.
TAGSEach file in its own labeled block — paths stay unambiguous across the whole bundle.
FENCESLanguage-typed fences, so syntax highlighting survives in whatever renders the reply.
<codebase project="my-app">
  <directory_structure>
    src/
    ├─ App.tsx
    ├─ hooks/use-theme.ts

  </directory_structure>

Separate tool · command line

Rather work in the terminal?

The browser app above needs no install. The FileConcat CLI is a separate package for piping directories straight into scripts — same structured output, printed to stdout.

PIPEPoint it at a folder; the bundle lands on stdout, ready to redirect or pipe.
PARSEOpt in with --parse to pull text out of PDF, DOCX, XLSX and more.
JSONA one-line --json summary for scripts: counts, bytes, elapsed.
$ npm install -g @fileconcat/cli

$ file-concat ./src \
    --parse pdf,docx -o ctx.xml
  Processing ./src
  Found 47 files · parsed 3 · skipped 2
  Done in 1.8s → ctx.xml
This is the command-line tool — not needed for the browser app.
